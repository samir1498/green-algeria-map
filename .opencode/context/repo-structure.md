## CI/CD Workflows

| Workflow | Trigger | Jobs |
|----------|---------|------|
| `ci.yml` | PR `frontend/**` | check → lint → knip → depcruise → test-unit → test-ui → test-it → coverage → build |
| `ci-backend.yml` | PR `backend-nestjs/**` | check → lint → knip → depcruise → test-unit → test-it → build |
| `ci-backend-springboot.yml` | PR `backend-springboot/**` | check → lint → arch → depgraph → test-unit → test-it → build |
| `ci-backend-go.yml` | PR `backend-go/**` | check → test-unit → test-it → build |
| `ci-e2e.yml` | workflow_dispatch | Playwright vs NestJS, Spring Boot, Go |
| `ci-bench.yml` | PR `benchmark/**` | typecheck + lint + knip + test |
| `ci-gate.yml` | PR any | Merge gate placeholder |
| `cd.yml` | push main `frontend/**` | Build + Deploy to Cloudflare Pages |
| `cd-backend-springboot.yml` | tag `backend-springboot-v*` | Native Docker image → GHCR |
| `cd-backend-go.yml` | tag `backend-go-v*` | Docker image → GHCR |

## Docker Compose

`docker-compose.yml`:
- **Shared infra**: postgres (18-alpine, port 5432), rustfs (S3 bucket, ports 9000/9001)
- **Profiles**: `[nestjs]` (8080), `[springboot]` (8081), `[go]` (8082)
- Per-backend DB: `greenalgeria_nestjs`, `greenalgeria_springboot`, `greenalgeria_go`

`config/docker-compose.dev.yml` — infra only (no profiles, used by dev scripts)
`config/docker-compose.e2e.yml` — E2E infra (separate container names)

## Conventions & Patterns

- **Clean Architecture**: domain/application/infrastructure layers (NestJS, Spring Boot)
- **Hexagonal**: Ports & Adapters pattern (Spring Boot)
- **CQRS**: CommandBus/QueryBus pattern (NestJS via `@nestjs/cqrs`, Spring Boot via custom bus, Go via service layer)
- **Repository Pattern**: Domain interfaces decoupled from ORM/store
- **Feature-sliced frontend**: `features/{name}/{api,hooks,components}/`
- **Dual-backend auth facade**: `features/auth/api/index.ts` selects impl via `VITE_API_BACKEND`
- **Session-as-router-context**: Auth state flows via TanStack Router context
- **Mapper pattern**: Domain ↔ ORM conversion in separate mapper classes
- **Status machines**: Domain-enforced state transitions (Zone status, DamageReport status)
- **Error normalization**: All backends return structured `{ statusCode, error, message, category }` JSON
